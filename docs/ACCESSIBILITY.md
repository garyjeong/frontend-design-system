# 접근성 가이드라인

## ARIA 속성

### `role` 속성
의미론적 역할 정의. HTML5에 적절한 의미가 있으면 사용하지 않음 (예: `<button>`에는 `role="button"` 불필요).

### `aria-label` 및 `aria-labelledby`
접근 가능한 이름 제공. 시각적 레이블이 없을 때 `aria-label`, 다른 요소 참조 시 `aria-labelledby` 사용.

### `aria-describedby`
추가 설명 제공. 오류 메시지, 도움말 등에 사용.

### `aria-modal`
모달임을 나타냄. 모달이 열려 있을 때 배경 콘텐츠 접근 불가를 알림.

## 키보드 네비게이션

### 포커스 관리
- **논리적인 포커스 순서**: DOM 순서와 시각적 순서 일치
- **포커스 트랩**: 모달, 드로어 등 오버레이 컴포넌트에서 `useFocusTrap` 훅 사용
- **포커스 복원**: 오버레이 닫힐 때 `useFocusRestore` 훅으로 포커스 복원
- **시각적 포커스**: `focus-visible` CSS 또는 `useFocusVisible` 훅 사용

### 키보드 인터랙션
- **Tab / Shift + Tab**: 다음/이전 포커스 가능한 요소로 이동
- **Enter / Space**: 기본 동작 활성화
- **화살표 키**: 탭, 메뉴, 슬라이더 등 관련 요소 그룹 내 이동

## 스크린 리더 지원

- **의미론적 HTML**: `<div>` 대신 `<button>`, `<nav>`, `<main>`, `<section>` 등 사용
- **이미지 대체 텍스트**: 의미 있는 이미지는 `alt` 속성 제공, 장식용은 `alt=""`
- **폼 컨트롤 레이블**: 모든 폼 컨트롤은 `<label>` 요소와 연결
- **상태 및 피드백**: 동적 콘텐츠는 `aria-live` 영역 사용

## 색상 및 대비

- WCAG 2.1 AA 레벨 이상 대비 비율 충족
- 색상만으로 정보 전달 금지 (아이콘이나 텍스트 설명 추가)

## 반응형 디자인

- 다양한 화면 크기에서 올바르게 렌더링
- 브라우저 텍스트 크기 최대 200% 확대 시에도 기능 유지

## 테스트 및 검증

- **jest-axe**: 컴포넌트 단위 테스트에 통합
